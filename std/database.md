# 数据库设计标准

## 表设计

0. 表字段设计必须符合三范式，必要时可反范式化（需说明理由）
1. 字符集使用 utf8mb4
2. 排序规则使用 utf8mb4_unicode_ci 或等效规则
3. 每张表必须有主键，优先使用自增整数或雪花 ID
4. 禁用外键，外键约束在应用层保证
5. 逻辑删除使用 deleted_at 字段，不用 is_deleted

## 命名规范

0. 表名使用 snake_case 复数形式：`users`、`order_items`
1. 字段名使用 snake_case
2. 主键名：`id`
3. 创建时间：`created_at`
4. 更新时间：`updated_at`
5. 删除时间：`deleted_at`
6. 状态字段：`status`，枚举值使用小写下划线：`pending_payment`

## 索引

0. 每张表除主键外，尽量建立索引
1. 查询条件字段必须建索引
2. 联合索引字段顺序按区分度从高到低排列
3. 唯一约束必须加，包括联合唯一约束
4. 索引名规范：`idx_表名_字段名`、`uk_表名_字段名`
5. 定期使用 EXPLAIN 检查慢查询

## 字段类型

0. 金额使用 DECIMAL，不用 FLOAT/DOUBLE
1. 时间使用 DATETIME/TIMESTAMP，不用字符串
2. 布尔使用 TINYINT(1) 或 BOOLEAN
3. JSON 字段仅用于非结构化、查询频率低的场景
4. 大文本使用 TEXT/BLOB，并单独建表关联
5. 枚举优先使用 TINYINT/SMALLINT + 应用层映射，MySQL 8.0+ 可用 ENUM

## 事务

0. 业务操作涉及多张表时必须使用事务
1. 事务范围尽量小，避免长事务
2. 事务中禁止调用外部服务
3. 只读查询不要加事务
4. 乐观锁使用 version 字段，悲观锁明确说明原因

## 查询规范

0. 禁止 SELECT *，只查询需要的字段
1. 禁止在循环中执行 SQL
2. 分页查询必须加排序条件
3. 深分页使用游标或延迟关联优化
4. 大表更新必须分批执行
5. JOIN 表数量不超过 3 个

## 变更管理

0. 所有 DDL 变更必须通过迁移脚本执行
1. 迁移脚本必须可回滚
2. 对数据库实体的更改要及时同步到 DTO、BO、EO、文档、message.properties、DDL
3. 加字段优先允许 NULL 或设置默认值，避免锁表
4. 删字段先标记废弃，两个版本后再删除

## 分库分表

0. 单表数据量超过 500 万或增长快时考虑分表
1. 分片键选择稳定、离散、查询频繁的字段
2. 分片后避免跨分片 JOIN
3. 全局唯一 ID 使用雪花算法或类似方案
