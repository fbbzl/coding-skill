# DevOps 标准

## CI/CD

0. 所有代码必须通过 CI 才能合并
1. CI 至少包含：lint、typecheck、test、build
2. 构建产物版本化，与 Git 提交关联
3. 部署流程自动化，减少人工干预
4. 发布前必须通过 staging 环境验证

## 环境管理

0. 环境变量集中管理，不硬编码
1. 区分开发、测试、预发布、生产环境
2. 生产配置与代码分离
3. 敏感配置使用密钥管理服务
4. 环境差异文档化

## 容器化

0. Dockerfile 使用多阶段构建
1. 基础镜像使用官方稳定版
2. 容器以非 root 用户运行
3. 镜像只包含必要文件
4. 镜像标签使用语义化版本或 Git SHA

## 部署策略

0. 蓝绿部署、金丝雀发布或滚动更新
1. 部署前自动备份数据库
2. 部署后自动健康检查
3. 失败自动回滚
4. 灰度期间监控核心指标

## 可观测性

0. 应用必须暴露健康检查端点
1. 关键路径必须记录结构化日志
2. 核心指标接入监控系统
3. 错误率和延迟必须告警
4. 分布式追踪覆盖关键链路

## 备份与恢复

0. 数据库定期备份
1. 备份数据定期恢复演练
2. 关键配置版本化管理
3. 灾难恢复文档保持最新
4. RTO/RPO 明确并测试

## 安全

0. 容器镜像定期扫描漏洞
1. 生产网络最小权限访问
2. 密钥使用专门管理工具
3. 审计日志保留至少 180 天
4. 变更必须有审批记录

## 文档

0. 部署流程文档化
1. 回滚流程文档化
2. 常见问题排查手册
3. 值班和升级路径明确
4. 架构图保持最新
