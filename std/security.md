# 安全标准

## 输入安全

0. 所有用户输入必须校验长度、类型、范围
1. SQL 使用参数化查询，禁止字符串拼接
2. 前端输出必须转义，防止 XSS
3. 文件上传必须校验类型、大小、内容，存储在隔离目录
4. 路径参数必须校验白名单，防止路径遍历

## 认证与授权

0. 密码必须使用 bcrypt/argon2 等慢哈希算法
1. Token 必须设置过期时间，支持刷新和吊销
2. 敏感操作需要二次认证
3. 权限校验遵循最小权限原则
4. 禁止前端做安全决策

## 敏感数据

0. 密钥、密码、Token 禁止硬编码在代码中
1. 配置文件中的敏感信息使用环境变量或密钥管理系统
2. 日志中禁止打印敏感字段
3. 数据库中敏感字段必须加密
4. 传输中敏感数据使用 TLS

## 会话安全

0. Cookie 设置 HttpOnly、Secure、SameSite
1. Session ID 必须随机且足够长
2. 登录失败增加验证码或限流
3. 异地登录、敏感操作发送通知
4. 登出立即失效服务端会话

## 常见漏洞防护

0. SQL 注入：参数化查询 / ORM，禁止字符串拼接
1. XSS：输出转义 + Content-Security-Policy
2. CSRF：Anti-CSRF Token + SameSite Cookie
3. SSRF：限制内网访问，校验 URL 协议和 IP
4. 反序列化漏洞：使用安全库，禁止不可信来源反序列化
5. 越权：每个接口校验资源归属

## 依赖安全

0. 使用依赖扫描工具检查已知漏洞（如 OWASP Dependency-Check、Snyk）
1. 定期更新关键依赖
2. 禁止引入不再维护的库
3. 私有依赖使用内部镜像

## 审计与监控

0. 记录所有认证、授权、敏感操作日志
1. 异常登录、高频操作触发告警
2. 保留安全日志至少 180 天
3. 定期进行安全审计
