# 前端开发标准

## 通用原则

0. 组件职责单一，一个组件只做一件事
1. 状态尽量靠近使用它的地方
2. 优先使用组合而非继承
3. 避免过度工程，不要为不存在的需求抽象
4. 保持一致的 UI 风格和交互体验

## 项目结构

0. 按功能域组织目录：`features/`、`components/`、`hooks/`、`utils/`、`services/`
1. 公共组件放在 `components/common/`
2. 业务组件放在对应 feature 目录下
3. 样式文件与组件就近放置
4. 路由配置集中管理

## 组件规范

0. 组件名 PascalCase，文件名与组件名一致
1. Props 必须声明类型
2. 受控组件优先于非受控组件
3. 副作用集中在 useEffect 或生命周期中
4. 避免在 render 中创建新函数或对象
5. 列表渲染必须提供稳定 key

## 状态管理

0. 本地状态用 useState / useReducer
1. 跨组件共享状态用 Context 或状态管理库
2. 服务端状态用专用数据获取库
3. 状态更新不可变
4. 避免在全局状态中存派生数据

## 性能

0. 图片懒加载和压缩
1. 路由懒加载
2. 大数据列表虚拟化
3. 避免不必要的重渲染
4. 使用 Web Workers 处理重计算
5. 首屏加载时间目标 < 3 秒

## 样式

0. 使用 CSS-in-JS 或原子化 CSS 方案
1. 避免全局样式污染
2. 响应式适配移动优先
3. 颜色、字体、间距使用设计系统 token
4. 支持暗色模式时统一管理变量

## 可访问性

0. 图片必须加 alt
1. 表单元素必须加 label
2. 支持键盘导航
3. 颜色对比度符合 WCAG 标准
4. 动态内容使用 ARIA 属性

## 国际化

0. 文本必须走 i18n 配置文件
1. 日期、时间、数字、货币按 locale 格式化
2. 从设计阶段考虑 RTL 布局

## 错误处理

0. 接口失败展示友好错误提示
1. 空状态、加载状态、错误状态、无权限状态必须处理
2. 全局错误边界捕获未处理异常
3. 用户输入实时校验并反馈

## 构建与部署

0. 静态资源 CDN 化
1. 环境变量按环境区分
2. 构建产物分析体积
3. Source Map 仅在非生产环境上传
