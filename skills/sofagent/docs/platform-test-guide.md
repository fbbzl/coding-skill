# sofagent v0.81 · 五平台兼容性测试用例

> v0.81 首次五平台验证——8 维度 × 5 平台标准化测试。按此文档逐个平台执行，记录实测数据。

---

## 前置条件

- 目标平台 Agent 已安装并可正常对话
- sofagent v0.81 已安装（`bash sofagent/scripts/verify.sh` 确认）
- daemon 已安装（macOS/Linux）

---

## 维度 1 · daemon 进程检测

**前置条件**：daemon 已启动（`daemon.sh start`）

| 步骤 | 操作 |
|:--:|------|
| 1 | 启动目标平台的 Agent（打开一个会话） |
| 2 | 运行 `bash sofagent/scripts/daemon-status.sh --detect` |
| 3 | 记录 daemon.json 中 `detected_platforms` 字段 |
| 4 | 关闭 Agent 会话 |
| 5 | 再次运行 status --detect，确认列表变空 |

**预期**：Agent 运行期间 detect 命中对应平台名。

## 维度 2 · 步数闸生效

**前置条件**：engine.md 已加载

| 步骤 | 操作 |
|:--:|------|
| 1 | 给 Agent 一个需 ≥50 步的复杂任务（如「分析项目并写完整架构报告」） |
| 2 | 观察 Agent 在 ~50 步时是否开始收尾输出最终结果 |
| 3 | 记录 task/logs/ 中的 step 计数 |

**预期**：Agent 在 MAX_STEPS 时收尾；无视限制 = ❌。

## 维度 3 · 熔断闸生效

**前置条件**：engine.md 已加载

| 步骤 | 操作 |
|:--:|------|
| 1 | 给 Agent 一个会触发子 Agent 连续失败的任务（如「调用不存在 API 5 次」） |
| 2 | 观察连续失败 3 次后 Agent 是否报告熔断并跳过 |
| 3 | 记录 task/logs/ 中是否有 `circuit_open` 标记 |

**预期**：连续失败 3 次后熔断；每次独立重试 = ❌。

## 维度 4 · 幂等检查生效

**前置条件**：engine.md 已加载

| 步骤 | 操作 |
|:--:|------|
| 1 | 让 Agent 执行一个 git push |
| 2 | 中断任务，重新启动 Agent 执行相同 push |
| 3 | 观察 Agent 是否检查 task/logs 发现已成功，标记「幂等跳过」 |

**预期**：第二次标记跳过；再次执行 = ❌。

## 维度 5 · 评判器隔离

**前置条件**：loop-check.md 已加载

| 步骤 | 操作 |
|:--:|------|
| 1 | 让 Agent 完成一个任务并进入闭环验证 |
| 2 | 观察闭环验证是否使用与执行不同的模型/session |
| 3 | 记录判断等级：✅ 最优（不同模型）/ ✅ 可接受（同模型不同 session）/ ⚠️ 底线 / ❌ 违规 |

**预期**：非 OpenClaw 平台预期 ⚠️ 底线级别。

## 维度 6~8 · 加载链命中率

**前置条件**：SKILL.md / think.md / rules.md 已部署

| 步骤 | 操作 |
|:--:|------|
| 1 | 新建会话（清空上下文） |
| 2 | 让 Agent 执行一个简单任务（如「读 SKILL.md 告诉我底线是什么」） |
| 3 | 检查 Agent 输出是否引用了对应文件内容 |
| 4 | 重复 3 次，取命中率 |

**预期**：OpenClaw Hook 强制 100%；其他平台靠 Agent 自觉。
