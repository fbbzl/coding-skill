## 概述

v0.81 做了 daemon 骨架 + 5 项治理加固，是「最诚实的概念验证」。然后做了一次双视角版本评审（GitHub 大神 + 科技公司技术负责人），挖出一批 P0/P1/P2 问题：

1. **治理加固约束级别混淆**：prompt 级软提醒写得像进程级硬拦截，误导技术负责人
2. **五平台能力矩阵全 ❓**：8 维度 × 5 平台，一个实测数据都没有
3. **文档重复度高**：已知局限在 4 份文档各写一遍，改一处要同步四处
4. **平台名不规范**：裸名 Hermes 与 NousResearch 大模型同名，搜索被稀释
5. **License 冗余**：文档不再需要 CC-BY-4.0，双许可增加企业采用门槛

v0.82 双线并行：**修复线**（填评审的坑）+ **验证线**（撕掉「未实测」标签）。同时做了 ROADMAP 结构重构、License 简化、SkillOpt 方法论引用。


## 第五部分：SkillOpt 方法论引用

### 要解决什么

ROADMAP v0.9 需要一个 Skill 训练方法论，但不想引入代码依赖（零依赖是 sofagent 的核心约束）。

### SkillOpt 是什么

微软研究院 2025 年的方法论（[arXiv:2505.16771](https://arxiv.org/abs/2505.16771)）——把 Skill 文档当模型的「外部状态」来训练。52 个 Skill × 4000 个任务，经过训练的文档在未见过的新任务上表现显著提升。

### sofagent vs SkillOpt

| 维度 | SkillOpt | sofagent |
|------|---------|---------|
| 目标 | 优化单条 Skill 的效果 | 治理多 Skill 的协作 |
| 依赖 | Python 训练管线 + 数据集 | 零依赖（纯 bash + MD） |
| 模型 | 可微优化器 | LLM 自身 |
| 反馈 | 自动评分函数 | Agent 行为 + task/logs |

### 借鉴的 3 个原则（纯方法论，不引入代码）

1. **文本学习率**：每次迭代只改 Skill 文档的一小部分（budget），不是推倒重写
2. **Held-out Gate**：修改后必须在未见过的新任务上通过，不能只对已知的过拟合
3. **拒绝缓冲区**：被否决的修改打上 `#rejected` 标签保留，不删——错误教训比成功经验更值钱

### 落盘

| 文件 | 改动 |
|------|------|
| `ROADMAP.md` | v0.9 新增 SkillOpt 段（是什么 / 4 步循环 / 对比表 / 3 原则 / 为什么不用代码） |
| `ARCHITECTURE.md` | 致谢表新增 Microsoft Research · SkillOpt 行 |
| `README.md` | 致谢段新增 SkillOpt |
