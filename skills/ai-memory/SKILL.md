# AI 记忆管理规范

## 定位

AI 记忆管理用于保存跨会话的项目上下文、编码约定、常见错误、最佳实践和错误教训，让 AI 编码助手在不同会话中保持一致性和准确性。

## 记忆类型

| 类型 | 内容示例 | 更新时机 |
|---|---|---|
| 项目上下文 | 技术栈、目录结构、构建命令、部署方式 | 项目初始化或重大变更 |
| 编码约定 | 命名规范、分层规则、禁止事项 | 团队规范更新或 CR 总结 |
| 常见错误 | 重复出现的 bug 模式、容易踩的坑 | 每次 bug 复盘 |
| 最佳实践 | 经过验证的实现模式、推荐库/工具 | 技术决策后 |
| 错误教训 | 已修复事故的根因和预防措施 | 事故复盘后 |
| 开放问题 | 当前未解决的技术债务或待决策项 | 每次迭代结束 |

## 存储位置

推荐放在项目内部，与代码一起版本化：

```
.ai-memory/
├── project-context.md
├── coding-conventions.md
├── common-mistakes.md
├── best-practices.md
├── lessons-learned.md
└── open-questions.md
```

如果记忆需要在多个项目间共享，可放在 `D:\AI\memory\` 下，并在 `opencode.jsonc` 中配置引用路径。

## 更新机制

- 每次重要会话结束前，由 opencode 或人工生成记忆摘要
- 摘要必须基于事实，禁止编造
- 更新时说明：来源、时间、相关文件、置信度
- 冲突时以最新、有证据、经过人工确认的记忆为准

## 记忆摘要模板

```markdown
## 条目标题

- 类型：项目上下文 / 编码约定 / 常见错误 / 最佳实践 / 错误教训 / 开放问题
- 来源：文件路径、PR、issue、会议记录
- 时间：YYYY-MM-DD
- 置信度：高 / 中 / 低
- 关联文件：相关代码或文档路径

内容描述...
```

## 检索与使用

- 新会话开始时，opencode 自动读取 `.ai-memory/` 下相关文件
- 按任务类型加载对应记忆：开发时读编码约定，排查 bug 时读常见错误
- 记忆不是 prompt 堆砌，只加载与当前任务相关的条目
- 当记忆与代码实际状态冲突时，以代码为准，并标记记忆需更新

## 隐私与安全

- 记忆中禁止包含 secrets、密码、token、私钥
- 禁止包含用户隐私数据、生产敏感数据
- 涉及安全漏洞的记忆只描述模式和修复方向，不暴露具体漏洞利用细节
- 定期审计 `.ai-memory/` 内容，清理过时或敏感信息

## 与 survey-corps 联动

- `dev` 在开发前读取项目上下文和编码约定
- `cr` 在审查后补充常见错误和最佳实践
- `qa` 在测试后补充 bug 模式和回归建议
- `dp` 在部署后补充部署教训和环境依赖

## 质量门禁

- [ ] 记忆内容有明确来源和证据
- [ ] 记忆分类清晰，便于检索
- [ ] 无敏感信息泄露
- [ ] 定期清理过时记忆
- [ ] 新成员能通过 `.ai-memory/` 快速理解项目关键约定
