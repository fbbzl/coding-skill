# 模板编写规范（Template Spec）

> 本文件定义 `templates/` 下所有图表模板的统一结构与质量基线。新增或修改模板时必须对齐本规范。范例参见 `templates/charts/bar-chart.md`、`templates/charts/line-chart.md`、`templates/charts/scatter-plot.md`。

## 为什么需要统一规范

模板是 anyviz 在「图表选择」与「代码生成」之间的桥梁。一个完整模板要同时回答四个问题：这张图什么时候用、数据长什么样、关键审美参数取什么值、设计上有哪些坑。缺任何一块，下游适配器（`adapters/`）就只能依赖库默认值，破坏跨图表一致性。

## 标准结构

每个模板必须包含以下章节，顺序固定：

```markdown
# 图表中文名（English Name）

## 适用场景
- 3-5 条要点，说明这张图回答什么问题
- 给出数据规模/类别数的建议区间（如「最佳类别数：2-15 个」「最佳数据点：50-10,000」）

## 数据格式
​```json
{ ...一个最小可用的输入数据示例... }
​```

## 美学参数（从 default.json 继承）
​```json
{ ...只列出该图表特有或需强调的参数，值必须与 default.json 对齐... }
​```

## 设计要点
1. **要点名**：具体规则，带可执行的数值（颜色 hex、px、opacity 等）
2. ...（4-6 条）

## 变体
### 变体名（English）
一句话说明何时用、与主图的差异
```

## 硬性要求

1. **审美参数写死、引用 default.json 真值**。所有颜色用 `default.json` 中的 hex（分类色板首色 `#4269d0`、增长 `#3ca951`、减少/对比 `#ff725c`、参考/中性 `#9696a0`、高亮强调 `#a463f2`、强调点描边 `#FFFFFF`）；线宽、点半径、opacity、圆角等取 `default.json` 对应字段值。不要发明新色值。

2. **色板选择遵循数据类型**。分类→categorical，顺序→sequential（单色相 Blues / 多色相 Viridis），发散→diverging，高亮→highlight。详见 `aesthetics/color.md`。

3. **数据格式给最小可用示例**，字段命名与同类模板保持一致（系列用 `series`/`name`/`values`，节点链接用 `nodes`/`links`/`source`/`target`/`value`，地理用 `regions`/`id`/`name`/`value`）。

4. **设计要点必须可执行**。「颜色协调」是废话；「连接线 #CCCCCC, 1px, opacity 0.5」才是规范。每条要点尽量带数值。

5. **无障碍兜底**。涉及仅靠颜色区分的图表，设计要点中必须提示纹理/形状/标签冗余编码，参见 `guides/accessibility.md`。

6. **变体不重复造图**。变体是主图的参数调整或扩展（如柱状图→堆叠/分组/棒棒糖），独立成型的图表应单独建模板并在 `chart-selection.md` 登记。

## 数值基线速查（来自 default.json）

| 元素 | 参数 | 值 |
|------|------|-----|
| 折线 | 线宽 / 插值 | 2.2px / monotone，圆角端点 |
| 散点 | 半径 / 描边 | 4.5px / 2.0px 白描边，opacity 0.75 |
| 折线数据点 | 半径 | 3.5px，opacity 1.0 |
| 高亮点 | 半径 / 描边 | 6.0px / 2.5px #333 |
| 异常值 | 半径 / 样式 | 4.5px / × 形，opacity 0.6 |
| 柱状 | 圆角 / 宽度比 | 5px / 0.65 |
| 面积 | 填充 opacity | 单层 0.12，堆叠 0.65 |
| 网格 | 线宽 / 颜色 / 虚线 | 0.5px / #E8E8E8 / [2,2]，仅 Y 轴 |
| 轴线 | 线宽 / 颜色 | 0.8px / #CCCCCC |
| 参考线 | 线宽 / 颜色 / 虚线 | 1.0px / #EE7B6F / [4,4]，opacity 0.7 |
| 趋势线 | 线宽 / 颜色 / 虚线 | 1.5px / #EE7B6F / [6,3]，opacity 0.6 |
