# 饼图 / 环形图（Pie & Donut Chart）

## 适用场景
- 展示各部分在整体中的占比
- 最佳扇区数：2-5 个
- ⚠️ 不适用于精确比较（人眼不擅长比较角度）

## 数据格式
```json
{
  "total": 1000,
  "slices": [
    { "name": "移动端", "value": 520 },
    { "name": "PC端", "value": 310 },
    { "name": "平板", "value": 120 },
    { "name": "其他", "value": 50 }
  ]
}
```

## 美学参数

```json
{
  "point": {},
  "bar": {},
  "annotation": {
    "data_label_offset": 6
  }
}
```

## 设计要点

1. **扇区排序**：从 12 点位置开始，顺时针按数值降序排列
2. **颜色分配**：按 categorical 色板顺序，相邻扇区避免相似色
3. **标签**：
   - 扇区 > 5%：显示名称和百分比
   - 扇区 ≤ 5%：仅显示名称，百分比合并或用指引线标注
4. **环形图优先**：环形图（中心留空）比实心饼图更易于阅读，且中心可用于显示总数
5. **慎用 3D 饼图**：3D 透视会扭曲面积比例，除非明确需要装饰效果
6. **无障碍**：扇区直接标注名称+百分比，不让读者仅靠颜色对照图例；相邻扇区色相间距足够，色盲可分

## 变体

### 环形图（Donut）
中心留空的饼图，中心区域可显示总数或关键指标，比实心饼图更易读，为默认推荐形态

### 玫瑰图 / 南丁格尔图（Rose / Polar Area）
扇区角度相等、半径表示数值，适合展示带周期性的类别（如 12 个月），但半径对面积的非线性放大需谨慎

### 半环形图（Half Donut / Gauge）
180° 半环，常用作仪表盘式的单指标占比展示（如完成率）

## 常见错误

- ❌ 饼图扇区超过 7 个（改用条形图）
- ❌ 用饼图展示变化（饼图仅适合展示静态占比）
- ❌ 用两个饼图做对比（人眼无法准确比较两个饼图）
