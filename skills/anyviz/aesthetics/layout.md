# 布局规则

## 核心原则

### 1. 数据-墨水比（Data-Ink Ratio）

遵循 Edward Tufte 的原则：最大化有意义的数据显示，最小化非数据元素。
- 默认不显示背景色块
- 网格线使用最浅的颜色和最细的线宽
- 去除不必要的边框和装饰

### 2. 视觉层次

信息应按重要性排列视觉层次：
1. 数据本身（最强视觉权重）
2. 轴和刻度（支持理解）
3. 标题和标签（提供上下文）
4. 网格和参考线（最弱视觉权重）

### 3. 图表尺寸与宽高比

- 默认宽高比 1.6:1（黄金比例近似值），适合大多数场景
- 时间序列数据可考虑 2:1（更宽的视角展示趋势）
- 地理数据保持原始地图投影比例
- 多图小倍数（Small Multiples）可以使用 1:1 正方形

### 4. 空白空间

足够的留白让图表"呼吸"，避免信息密度过高：
- 图表区占画布面积的 65-75%
- 标签和标题占 15-20%
- 图例占 5-10%

## 自然语言布局指令映射

| 自然语言                     | 美学参数调整                              |
|-----------------------------|-----------------------------------------|
| "留白多一些"                  | 各边距 ×1.3，元素间距 ×1.2                |
| "紧凑布局"                    | 各边距 ×0.75，元素间距 ×0.8               |
| "宽屏展示"                    | 宽高比 → 2.0                             |
| "正方形比例"                  | 宽高比 → 1.0                             |
| "适合移动端"                  | 应用 mobile 断点的 font_scale 和 margin_scale |

## 网格与多图布局

多个图表在同一画布或仪表盘中排布时，网格系统和对齐规则决定了整体视觉秩序。

### 仪表盘网格列数建议

根据内容复杂度和屏幕宽度，推荐不同的列数。`default.json` 中的 `responsive.breakpoints` 定义了三个标准断点。

| 屏幕宽度 | 设备 | 推荐列数 | 示例布局 | 理由 |
|----------|------|---------|---------|------|
| ≤ 640px | 手机 | 1 | 单列堆叠 | 减少滚动横向分散注意 |
| 641-1024px | 平板 | 2 | 两列网格 | 平衡信息密度与阅读流畅性 |
| > 1024px | 桌面 | 3-4 | 三列或四列网格 | 充分利用屏幕空间，但避免超过 4 列（认知负荷过高） |

**超过 4 列的风险**：
- 每个图表过小，难以辨别细节
- 视线需要频繁左右扫动，阅读体验差
- 建议改用标签页或滚动容器分页展示

### 网格对齐基线

所有图表在网格中应遵循统一的对齐基线，确保标题、图表区、图例的上下左右对齐。

```
标题基线对齐：所有图表的 h1 在同一高度
┌───────────────────────┐  ┌───────────────────────┐
│ 2024 年销售额          │  │ 2024 年利润            │ ← h1 基线
├───────────────────────┤  ├───────────────────────┤
│                       │  │                       │
│        [图表 A]       │  │        [图表 B]       │ ← 图表区对齐
│                       │  │                       │
└───────────────────────┘  └───────────────────────┘
```

**实现方法**：
- 使用 CSS Grid 或 Flexbox 的 `align-items: start`
- 所有图表容器设置相同的 padding 和 margin
- 使用 `default.json` 的统一 `chart_margin` 值

### 多图共享坐标轴与图例

当多个图表展示相关数据时，可以节省空间并增强对比。

#### 共享 Y 轴

**适用**：展示同一数据在不同维度的分解（如产品 A/B/C 的月度销售）

```
X 轴共享 ←─────────────────────────────────────────→
          1月   2月   3月   4月   5月   6月

产品 A：┌─────────────────────────────────┐  
        │ [图表数据]                      │
        └─────────────────────────────────┘

产品 B：┌─────────────────────────────────┐  ← Y 轴标度一致
        │ [图表数据]                      │      便于对比
        └─────────────────────────────────┘

产品 C：┌─────────────────────────────────┐
        │ [图表数据]                      │
        └─────────────────────────────────┘
```

**关键规则**：
- Y 轴范围 (0 - max) 必须一致
- 每列只保留左侧或右侧的一个 Y 轴标签，其他省略
- 同一行的所有图表宽度相同

#### 共享图例

多个相同类型的图表可共享一个图例，放在上方或左侧。

```
           销售额    销售量    毛利
┌─────────────────────────────────┐  ← 共享图例
│ ■ 华东  ■ 华中  ■ 华西         │
└─────────────────────────────────┘

┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
│ 销售额（分地区）  │  │ 销售量（分地区）  │  │ 毛利（分地区）   │
│ [柱状图]         │  │ [柱状图]         │  │ [柱状图]         │
└──────────────────┘  └──────────────────┘  └──────────────────┘
```

**优势**：
- 减少重复视觉元素，突出数据本身
- 图表容器可更紧凑

**风险**：
- 图表个数 > 4 时，共享图例的关联关系可能不够直观
- 用户可能误认为图例只属于第一个图表

### 黄金分割与网格基础单位

遵循黄金比例 1:1.618 或近似值 1.6:1 有助于视觉和谐。

```
标准图表宽高比：1.6:1（default.json: "aspect_ratio": { "default": 1.6 }）
→ 若容器宽 640px，图表高应为 400px
→ 多图网格时，按此比例规划每个网格单元大小
```

**网格基础单位（8px 系统）**：
- `chart_margin`：48/36/56/56（单位 px）
- `multi_chart_gap`：28px（可按 8 的倍数调整：24、32、40）
- 所有间距应为 4 或 8 的倍数，便于精确对齐

## 边距与响应式

图表的边距（margin）和间距在不同屏幕上需要适应性调整，以保持信息密度与可读性的平衡。

### 标准边距规范与断点应用

`default.json` 中定义的边距和响应式参数：

```json
"spacing": {
  "chart_margin": { "top": 48, "right": 36, "bottom": 56, "left": 56 },
  "chart_margin_small": { "top": 32, "right": 24, "bottom": 40, "left": 48 }
},
"responsive": {
  "breakpoints": {
    "mobile": { "max_width_px": 640, "font_scale": 0.85, "margin_scale": 0.7 },
    "tablet": { "max_width_px": 1024, "font_scale": 0.92, "margin_scale": 0.85 },
    "desktop": { "max_width_px": 999999, "font_scale": 1.0, "margin_scale": 1.0 }
  }
}
```

#### 三断点的应用规则

**移动端（≤ 640px）**：
```
font_scale: 0.85   → 所有字号 × 0.85（16px → 13.6px ≈ 14px）
margin_scale: 0.7  → 所有边距 × 0.7
  chart_margin: (48×0.7=33, 36×0.7=25, 56×0.7=39, 56×0.7=39)
  实际应用: (top: 33, right: 25, bottom: 39, left: 39)
```

**平板（641-1024px）**：
```
font_scale: 0.92   → 字号 × 0.92（中间过渡）
margin_scale: 0.85 → 边距 × 0.85
  chart_margin: (48×0.85=41, 36×0.85=31, 56×0.85=48, 56×0.85=48)
```

**桌面（> 1024px）**：
```
font_scale: 1.0    → 完整字号（不缩放）
margin_scale: 1.0  → 完整边距（不缩放）
  chart_margin: (top: 48, right: 36, bottom: 56, left: 56)
```

#### chart_margin 和 chart_margin_small 的选择

- **chart_margin**（标准）：默认用于大型仪表盘、报告中的主要图表
- **chart_margin_small**（紧凑）：用于空间受限场景
  - 多图表在同一行（> 3 个）
  - 移动端纵向滚动
  - 嵌入式小图表

**判断标准**：
```
若 (容器宽度 / 图表个数) < 400px  → 使用 chart_margin_small
若 (容器宽度 / 图表个数) ≥ 400px  → 使用 chart_margin
```

### 响应式实现的实际调整

**CSS 媒体查询示例**：

```css
/* 桌面 */
.chart-container {
  font-size: 16px;
  padding: 48px 36px 56px 56px;
}

/* 平板 */
@media (max-width: 1024px) {
  .chart-container {
    font-size: 14.7px;  /* 16 × 0.92 */
    padding: 41px 31px 48px 48px;
  }
}

/* 移动端 */
@media (max-width: 640px) {
  .chart-container {
    font-size: 13.6px;  /* 16 × 0.85 */
    padding: 33px 25px 39px 39px;
    column: 1;  /* 单列布局 */
  }
}
```

### 何时改用 chart_margin_small

多个判断条件触发 chart_margin_small 的使用：

| 触发条件 | 示例 | 处理 |
|---------|------|------|
| 移动端（≤640px） | 单列堆叠图表 | 自动应用 margin_small + margin_scale 0.7 |
| 紧凑仪表盘（2-3 列） | 3×2 网格 6 个卡片 | 手动设置 margin_small |
| 嵌入式小窗口 | 侧边栏图表 | 明确指定 margin_small |
| 对比布局 | 两个相邻柱状图对比 | 可选，取决于空间 |

## 视觉权重与焦点

在数据可视化中，通过大小、颜色、位置、留白等视觉维度来引导用户注意力，确保关键信息被优先感知。

### 四种引导视线的视觉维度

#### 1. 大小（Size）

大的元素吸引视线。在仪表盘中应用：

```
KPI 数值字号：20px（相比其他文字 16px 更突出）
销售额占比：图表宽度占 70%，参考信息占 30%
```

**规则**：
- 最关键数据 ≤ 1.5× 次要数据的大小（不要极端对比，易显廉价）
- 留白越多的地方，元素越吸引注意

#### 2. 颜色（Color）

饱和度高、对比度强的颜色吸引视线。

```
目标值：使用 highlight.primary (#4269d0) 或 highlight.comparison (#ff725c)
次要数据：降低饱和度或增加透明度（opacity: 0.3）
参考线：灰色、淡色，opacity < 0.5
```

**规则**：
- 一张图表最多 3 个高饱和颜色
- 超过 3 种时，其余用渐进式的灰度或淡化版本
- 动态高亮时，非焦点数据改为 #CCCCCC 或 opacity 0.2

#### 3. 位置（Position）

用户的自然阅读路线：
- 左上 → 右上（首眼）
- 左中 → 右中（主视区）
- 底部（最后注意）

```
布局建议：
标题（左上）
 ↓
主图表（中央）
 ↓
参考数据/图例（右侧或底部）
```

#### 4. 留白（Whitespace）

充分的留白让信息"呼吸"，周围越空白的区域越显重要。

```
仪表盘示例：
┌─────────────────────────────────┐
│  2024 年销售数据                 │  ← 标题留白充足，视觉突出
│                                 │
│  ┌─────────────────────────────┐│
│  │                             ││
│  │        销售额: ¥2.5M        ││  ← 主 KPI 被大量留白包围，极其突出
│  │                             ││
│  └─────────────────────────────┘│
│                                 │
│  同比增长 8.5%    环比增长 2.1%  │  ← 次要指标紧凑排列
└─────────────────────────────────┘
```

### Z 字与 F 字阅读模式

根据内容类型，用户的视线扫动方式不同。

#### Z 字阅读（Zigzag Reading）

适用于**横向对比类**图表（多柱、多线并列）。

```
点 1（左上）          点 2（右上）
        ╲        ╱
         ╲      ╱
          ╲    ╱
           ╲  ╱
点 3（左下）        点 4（右下）
```

**优化建议**：
- 最重要信息放在左上和右上（图表标题、高亮点）
- 关键数据标注在右侧或上方（终点、极值）
- 图例放在上方，便于首次扫描

#### F 字阅读（Top-Left Bias）

适用于**纵向流程类**图表（时间序列、排名列表）。

```
点 1 ─────────────── 点 2（全宽扫描，寻找关键词）
│
点 3（竖向扫描）
│
点 4
│
点 5（可能不会看到）
```

**优化建议**：
- 标题和第一条数据应包含最关键信息
- X 轴标签放在左侧或顶部
- Y 轴标签（颜色、分类）尽量靠上
- 降序排列（最大的在顶部），符合用户期望

### 突出单一关键数据的技法

当报告核心是某一个 KPI 时，用以下技法强调：

| 技法 | 应用 | 示例 |
|------|------|------|
| 加大字号 + 字重 | KPI 主数值 | 20px, 700 weight |
| 色彩突出 | 用 highlight.primary 包围 | 蓝色背景卡片 |
| 留白隔离 | 单独面板，周围大量空白 | 页面上方 40% 空间只显示一个数值 |
| 高对比 | 数值与背景、周边数据的对比极高 | 黑底白字数值，其他元素灰色 |
| 动画吸引 | 页面加载时数值从 0 跳到目标值 | 脉冲效果（谨慎：尊重 prefers-reduced-motion） |
| 对标线 | 目标值、同比值作为参考线 | 虚线叠加，数值标注 |

**完整示例（销售仪表盘）**：

```
┌──────────────────────────────────────────────────┐
│                                                  │
│                                                  │
│              销售额（本月）                       │
│              ¥ 2,500,000                         │  ← 最大字号、蓝色、居中
│                                                  │
│              ↑ 8.5% vs 目标 ¥2,300,000            │  ← 小字、绿色增长箭头、对标
│                                                  │
│                                                  │
└──────────────────────────────────────────────────┘
（以下为次要内容，较小字号、较紧凑布局）
┌──────────────────┐  ┌──────────────────┐
│ 区域排名          │  │ 产品排名          │
│ 华东 ¥1.2M       │  │ 产品 A ¥800K     │
└──────────────────┘  └──────────────────┘
```

### 检查清单

```
□ 最关键信息是否通过大小、颜色、位置组合强调？
□ 用户视线路径是否清晰（Z 字或 F 字）？
□ 是否用足够留白隔离重点信息？
□ 高亮时是否有超过 3 种高饱和颜色？
□ 参考线/辅助信息是否降低了视觉权重（灰化或透明化）？
□ 多个竞争焦点吗？如有，是否有明确的优先级排序？
□ 打印/灰度模式下关键信息是否仍可区分？
```
