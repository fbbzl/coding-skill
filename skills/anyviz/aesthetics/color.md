# 色彩规则

色彩方案受 Observable Plot 现代设计语言启发，结合 ColorBrewer、Viridis 等学术色彩方案的实践。

## 核心原则

### 1. 色板与数据类型匹配

| 数据类型 | 适用色板         | 示例场景                    |
|----------|------------------|----------------------------|
| 分类     | categorical      | 产品类别、地区、部门         |
| 顺序     | sequential       | 温度、密度、收入等级         |
| 发散     | diverging        | 同比变化率、偏差值、净推荐值  |
| 高亮     | highlight        | 目标值、预警线、基准线       |

### 2. 色盲友好

约 8% 的男性和 0.5% 的女性有色觉障碍。默认色板应满足：
- 红绿盲（Protanopia/Deuteranopia）：避免仅靠红-绿区分
- 蓝黄盲（Tritanopia）：较少见，但高对比度的蓝-黄也应避免

默认 categorical 色板经过 ColorBrewer 验证，10 种颜色均可被色盲人群区分。

### 3. 语义一致性

- 正增长/好 → 蓝或绿色系
- 负增长/差 → 红或橙色系
- 中性/参考 → 灰色系
- 高亮/警告 → 红色系（仅用于需要引起注意的信息）

### 4. 颜色数量控制

- 分类色板建议 ≤ 7 种颜色（超过时考虑合并小类为"其他"）
- 顺序色板建议 ≥ 5 个梯度（太少无法区分）
- 一张图表中高度饱和的颜色不超过 3 种

## 自然语言颜色指令映射

当调用者提出颜色相关的自然语言要求时，按以下映射调整：

| 自然语言                     | 美学参数调整                                      |
|-----------------------------|--------------------------------------------------|
| "颜色更活泼/鲜艳"             | 色板饱和度 ×1.3，使用 multi_hue 替代 single_hue     |
| "颜色素雅/低调"               | 饱和度 ×0.7，亮度 +10%                             |
| "深色主题/暗色背景"           | 背景 #1a1a1a，文字 #f0f0f0，色板亮度整体 +15%       |
| "使用公司品牌色"              | 替换 categorical[0] 为首要品牌色，其余用品牌色衍生   |
| "暖色调"                     | 限制色相范围在 0-60°（红-橙-黄）                    |
| "冷色调"                     | 限制色相范围在 180-270°（青-蓝-紫）                 |
| "对比度更强"                  | 色板相邻颜色色相间距至少 30°                        |
| "柔和对比较弱"                | 色板颜色色相间距 ≤ 15°，整体降低饱和度               |
| "突出 {类别}"                | 目标类别使用 highlight.primary，其余类别 opacity: 0.3 |

## 色板生成与扩展

当类别数量超过色板上限（default.json 中 categorical 最多 10 色）时，需要进行策略性处理。

### 超过色板上限的处理策略

| 策略 | 适用场景 | 实现 |
|------|---------|------|
| 合并"其他" | 类别 > 10，但主要关注前 8-9 类 | 将低频/次要类别合并为灰色"其他"，标注省略项数 |
| 明度梯度衍生 | 颜色有层级关系（年度>季度>月度） | 保持色相不变，从亮到暗衍生 5-7 级（如天蓝 → 深蓝） |
| 循环 + 纹理 | 类别众多但视觉不能模糊 | 色板循环使用，用不同线型/点型/填充纹理区分相同色的数据 |
| 饱和度梯度 | 类别有优先级 | 主要类别全饱和，次要类别降低饱和度 20-30% |

**推荐限制**：
- 单图表颜色数 ≤ 7（易认知）
- 必须超过 7 时，总数 ≤ 12（否则用分组/小倍数替代）
- 饼/环图 ≤ 5 色（超过则考虑折线图或堆叠柱状图）

### 从品牌色生成协调色板

当用户指定单一品牌色（如公司主色 #4269d0）时，可用以下色相方案生成配套的分类色板。

#### 互补色方案（适合对比强、数据两级对立）

| 角色 | 色相关系 | 颜色 | 使用场景 |
|------|---------|------|---------|
| 主色 | 0° | 品牌色本身 | 第一重要类别 |
| 补色 | 180° | 原色的相反色 | 第二重要 / 对比类别 |
| 中间色1 | 90° | 品牌色顺时针旋转 90° | 中性 / 过渡 |
| 中间色2 | 270° | 品牌色逆时针旋转 90° | 中性 / 过渡 |
| 衍生1 | -30° | 品牌色 ±30° 的近似色 | 细分类别 |
| 衍生2 | +30° | 品牌色 ±30° 的近似色 | 细分类别 |

**示例**（品牌蓝 #4269d0 ≈ 217°）：
- 互补（37°）：橙色 #F5A623
- 近似（247°）：紫色 #A463F2
- 近似（187°）：青色 #6CC5B0

#### 类比色方案（适合数据相近/层级递进）

相邻 30° 范围内的色相，视觉上和谐统一，适合展示序列或相关数据。

```
品牌色（蓝）→ 青蓝 → 青 → 青绿 → 绿（色相 217° ± 30°）
用途：产品线纵向对比、季度递进、区域嵌套层级
```

#### 三分色相方案（适合三类数据的等权对比）

品牌色 + 两个相隔 120° 的颜色。

```
蓝（217°） + 红/橙（37°） + 绿/黄（97°）
均衡三足鼎立，视觉权重相当
```

**实现提示**：
- 在色相环上标记品牌色，按需旋转 30°、90°、120°、180° 等关键角度
- 保持饱和度和亮度一致，只变色相，确保视觉权重可比
- 衍生色经过 ColorBrewer 或 Accessible Colors 工具验证无障碍性

## 透明度与叠加

多层数据叠加时，透明度是控制视觉权重和防止过度遮挡的关键手段。`default.json` 中已定义各场景的标准 opacity 值。

### 各场景的透明度规范

| 场景 | opacity | 用途 | 示例 |
|------|---------|------|------|
| 面积填充（单层） | 0.12 | 背景烘托，不抢焦点 | 面积图背景 |
| 堆叠面积（多层） | 0.65 | 显示完整堆叠形状，适度透明避免完全遮挡 | 堆叠面积图 |
| 渐变上端 | 0.25 | 面积渐变效果的上边界 | 拓扑面积图 |
| 渐变下端 | 0.0 | 面积渐变效果的下边界 | 拓扑面积图 |
| 散点（单类） | 0.75 | 显示个体点，允许部分重叠 | 散点图基础 |
| 散点（高密度区） | 0.3-0.5 | 密集点聚集区自然加深 | 密度热点自动生成 |
| 散点（低密度区） | 0.75 | 稀疏点保持清晰 | 离群点 |
| 非高亮数据 | 0.25 | 减弱非焦点数据 | 交互高亮时的背景 |
| 趋势线/参考线 | 0.6-0.8 | 辅助线不抢主数据焦点 | 辅助线叠加 |

### 多层叠加的视觉混色注意

当多个半透明颜色叠加时，会产生视觉混色效果，可能与预期颜色差异大。

**常见问题与解决**：

1. **蓝 + 红 = 紫？** 在叠加区域，应使用柔和的紫色（如 #C7ADEA）作为预期结果，或改用堆叠而非叠加
2. **黄色失效** 浅黄色在任何背景上叠加后都易丢失，优先降低透明度或改用更深的金黄色（#E8B33F）
3. **灰色中性** 灰色 #999999 在任何彩色上叠加仍保持中性感，适合作"无类别"或"未分类"的通用底色

**最佳实践**：
- 3 层以上叠加：用堆叠而非透明度叠加，避免视觉混乱
- 需要叠加时，测试目标两种颜色的叠加结果，预先调整透明度
- 用 `rgba()` 显式指定，不依赖库默认值

## 深色主题配色

当用户指定"深色背景"或"深色主题"时，应按以下调整规范转换色板。

### 深色背景下的色板调整

| 属性 | 亮色主题 | 深色主题 | 调整规则 |
|------|---------|---------|---------|
| 背景 | #FFFFFF | #1A1A1A | 背景 → 深灰 |
| 文本主色 | #1A1A1A | #E8E8E8 | 亮度反转 + 5% 缓和 |
| 文本次色 | #555555 | #AAAAAA | 次色提亮 ≈ 35% |
| 文本三级 | #666666 | #999999 | 三级提亮 ≈ 35% |
| 文本弱化 | #888888 | #777777 | 弱化色略提亮 |
| 网格线 | #E8E8E8 | #333333 | 深灰边界 |
| 边框 | #D0D0D0 | #444444 | 深灰边框 |

### 数据色的亮度调整

在深色背景（#1A1A1A）上，categorical 色板应整体**提亮 15%**（Lightness +15）。

```
亮色版（#FFFFFF 背景）：
#4269d0（蓝）、#3ca951（绿）、#ff725c（红橙）

深色版（#1A1A1A 背景）：
#5B8DE6（提亮的蓝）、#4FC15D（提亮的绿）、#FF8976（提亮的红橙）
```

**调整方法**（HSL 色彩空间）：
- 保持色相、饱和度不变
- Lightness 提升 15 个百分点
- 若提亮后饱和度显得过低，可同时提升饱和度 5-10%

### 深色下的饱和度与对比

深色背景会强化彩色的视觉冲击，容易显得刺眼。处理方式：

1. **降低饱和度 10%**：高饱和的鲜艳色在深色上易疲劳，适度降饱和度使其柔和
2. **配搭明度对比**：确保数据色与 #1A1A1A 的亮度差 ≥ 4:1（可比白背景更严格）
3. **用透明度代替饱和度**：若需要柔和，优先降低 opacity 而非饱和度

### 深色主题的特殊元素

| 元素 | 深色主题调整 | 示例 |
|------|------------|------|
| 高亮区域 | opacity 改为 0.12（从亮色的 0.08） | 背景烘托框 |
| 参考线 | 颜色 → #FF9999（提亮），opacity 0.5（略降） | 目标线、预警线 |
| Tooltip 背景 | #2A2A2A（深灰）而非白 | 浮层背景 |
| Tooltip 文字 | #F0F0F0（亮白）而非 #1A1A1A | 浮层文字 |
| 网格虚线 | 保持 #333333 或改为 #444444 | 细节线 |

### 深色主题的完整调整示例

```json
{
  "background": {
    "chart": "#1A1A1A",
    "page": "#0F0F0F",
    "grid": "#333333",
    "border": "#444444"
  },
  "color": {
    "categorical": [
      "#5B8DE6",  // 蓝 +15% lightness
      "#4FC15D",  // 绿 +15% lightness
      "#FF8976",  // 红橙 +15% lightness
      // ... 其余颜色同样处理
    ]
  },
  "text": {
    "primary": "#E8E8E8",
    "secondary": "#AAAAAA",
    "tertiary": "#999999",
    "muted": "#777777"
  },
  "annotation": {
    "tooltip_background": "#2A2A2A",
    "tooltip_border": "#555555"
  }
}
```

**验证清单**：
- 文本与背景对比度是否 ≥ 4.5:1？
- 数据色与深色背景对比度是否 ≥ 3:1？
- 是否避免了过度饱和导致的视觉刺眼？
- 所有重要信息是否在灰度模式下仍可区分？
