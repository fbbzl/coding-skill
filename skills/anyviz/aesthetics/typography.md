# 排版规则

## 核心原则

### 1. 字体层级清晰

从最重要的信息（标题）到最不重要的信息（脚注），字号应形成一个清晰的递减层级。
相邻层级的字号差至少 1-2px，确保视觉上有明显区分。

### 2. 字体选择

- **无衬线（Sans-serif）**：默认选择，适合屏幕阅读和数据可视化
- **衬线（Serif）**：适合正式报告、印刷出版
- **等宽（Monospace）**：仅用于代码、数值表格

### 3. 中文字体适配

在中英文混排时，优先使用系统原生中文字体以确保渲染质量：
- macOS: PingFang SC
- Windows: Microsoft YaHei
- Linux: Noto Sans CJK SC

通过在 font-family 中按优先级列出所有平台的字体来实现跨平台一致性。

### 4. 文字方向

- 图表标题：水平，左对齐或居中
- Y 轴标题：垂直或水平（如果空间允许，水平更易阅读）
- X 轴刻度标签：水平，若拥挤则旋转至 30-45°
- 图例标签：水平，在可用空间内排列

## 自然语言排版指令映射

| 自然语言                     | 美学参数调整                                |
|-----------------------------|-------------------------------------------|
| "字体大一点"                  | 所有层级字号 ×1.15                          |
| "字体小一点"                  | 所有层级字号 ×0.85                          |
| "标题更突出"                  | h1 字号 +2px，字重 → 700                   |
| "紧凑一些"                    | 行高 ×0.85，各间距 ×0.8                     |
| "正式/学术风格"               | 字体族 → serif，字号统一 +1px               |
| "现代/科技风格"               | 字体族 → sans-serif，字间距 -0.02em         |

## 数字排版

数字在数据可视化中极为关键，需要特殊的排版规范确保易读和对齐。

### 等宽数字（Tabular Figures）

在表格、仪表盘、数据标签中，数字应使用**等宽字体**（tabular-nums），确保数字宽度一致，便于竖向对齐。

```css
/* CSS 示例 */
.data-label {
  font-variant-numeric: tabular-nums;  /* 或 font-feature-settings: "tnum"; */
}
```

| 应用场景 | 示例 | 效果 |
|---------|------|------|
| 表格数据列 | 1000, 999, 100 | 竖向对齐，一眼看清大小关系 |
| 仪表盘数字 | 价格、销量、百分比 | 数字均匀排列，视觉稳定 |
| 轴刻度标签 | 2000, 1500, 1000 | 坐标刻度整齐对齐 |
| 数据标签（柱状图） | 每根柱子顶部数字 | 便于目测数值大小 |

### 千分位分隔与缩写

根据 `default.json` 的 `number_format` 规范，不同数据类型采用不同显示形式。

#### 千分位处理

```
default.json: "use_thousands_separator": true
```

- 数值 ≥ 1,000：使用千分位分隔符
  - 中文数据：1,234,567（逗号）或 1 234 567（不建议，易混淆）
  - 国际化：1,234,567（美式）或 1.234.567（欧式）
  - **建议统一使用逗号** #000，符合多数商业软件习惯

- **特殊情况**：金融数据保留千分位，科学数据在紧凑布局下可省略

#### 大数缩写

当数字超过百万时，应根据 `scientific_notation_threshold: 1000000` 和大小采用缩写。

| 规模 | 缩写 | 示例 | 场景 |
|------|------|------|------|
| 千 | K | 1.2K、45K | 社交媒体、粉丝数 |
| 万 | W（仅中文） | 12.5万、450万 | 中文报告、销售额 |
| 百万 | M | 1.5M、12M | 国际化、全球数据 |
| 亿 | 亿（仅中文） | 1.2亿、50亿 | 中文财务报告 |
| 十亿 | B | 2.5B、100B | 国际化、巨大规模 |

**缩写规则**：
- 保留 1-2 位小数（根据精度需求）
- 与 `decimal_places` 配置保持协调
- 为了空间和易读性，优先用缩写而非科学计数法（1.2M vs 1.2e6）

#### 小数位一致性

同一图表中相同数据类型的小数位应保持一致。

```
default.json: "decimal_places": 1, "percentage_decimals": 1
```

| 数据类型 | 小数位 | 示例 | 规则 |
|---------|--------|------|------|
| 通用数值 | 1 | 1234.5、99.8 | 默认，适合大多数商业数据 |
| 百分比 | 1 | 45.3%、12.0% | 不要超过 1 位，避免虚假精度感 |
| 小数货币 | 2 | ¥1,234.56 | 符合财务规范 |
| 比率 | 2-3 | 0.45、1.235 | 科学数据，根据需求调整 |
| 库存、计数 | 0 | 1234、999 | 整数，禁止小数 |

**验证清单**：
```
□ 表格和标签中数字是否使用了 tabular-nums？
□ 千分位分隔符是否一致？（都用逗号或都不用）
□ 是否采用了合理的大数缩写？
□ 同类数据的小数位是否一致？
```

## 标签防重叠

数据标签、轴刻度标签在空间拥挤时易重叠，影响可读性。此章节提供递进式解决方案。

### 问题与解决策略

**优先级顺序**（推荐按此顺序尝试）：

#### 1. 刻度标签旋转

X 轴标签拥挤时，逐步增加旋转角度。

```
default.json: "x_axis": {
  "tick_rotation": 0,
  "max_tick_rotation": 45,
  "tick_count_suggestion": 6
}
```

| 旋转角度 | 适用场景 | 可读性 |
|----------|---------|--------|
| 0° | 标签短且不重叠 | 最高 |
| 30° | 标签中等长度，轻微重叠 | 高 |
| 45° | 标签较长或数量众多 | 可接受 |
| -45°/90° | 特殊情况（Y 轴标签或竖排） | 中等 |

**规则**：不建议超过 45°（可读性急剧下降）。

#### 2. 刻度抽稀

减少显示的刻度标签数量，保留关键位置（第一、中间、最后）。

```
选择性显示：
完整：Jan, Feb, Mar, Apr, May, Jun ... 每月
抽稀 50%：Jan, Mar, May, Jul, Sep, Nov（每两月一个）
抽稀 80%：Jan, Apr, Jul, Oct（每季度）
```

**规则**：
- 最少保留 4-5 个主刻度（过少无法估值）
- 次刻度可完全隐藏，改用网格线指示

#### 3. 标签截断与省略号

标签过长时，截断后添加省略号（…），用 Tooltip 提供完整信息。

```
原文：华东地区销售部（2024 年第一季度）
截断：华东地区销售…
Tooltip：华东地区销售部（2024 年第一季度）
```

**实现规则**：
- CSS 方案：`text-overflow: ellipsis; overflow: hidden; white-space: nowrap;`
- 字符限制：15-20 字为宜（汉字 7-10）
- 必须配套 Tooltip 或表格详情视图

#### 4. 力导向避让（Collision Detection）

对于散点图、气泡图等自由标注的数据点，使用力导向算法（如 D3 的 force-simulation）将重叠的标签推开，连线指向数据点。

```javascript
// D3 示例（伪代码）
d3.forceSimulation(labels)
  .force("collide", d3.forceCollide(labelWidth/2 + padding))
  .force("x", d3.forceX(label.x))
  .force("y", d3.forceY(label.y))
```

**适用场景**：
- 散点数据点数 < 50（过多力导向效果差）
- 交互式图表（可接受线条视觉复杂度）

**不适用**：
- 静态图表（连线过多显得杂乱）
- 打印/PDF 输出（线条易断裂）

#### 5. 直接标注 vs 图例

**转换策略**：
- 标签过多 → 改用图例 + 颜色编码，减少直接标注
- 图例太长 → 改用分组图例或两列布局
- 两者都复杂 → 提供可交互的悬停 Tooltip

| 方案 | 标签数 | 视觉复杂度 | 交互性 | 推荐 |
|------|--------|-----------|--------|------|
| 直接标注 | ≤ 5 | 低 | 无 | 清晰简洁 |
| 混合标注 | 5-10 | 中 | 低 | 平衡方案 |
| 图例 + 颜色 | 10+ | 高 | 高 | 复杂数据 |

### 具体图表类型的防重叠规则

| 图表类型 | 常见重叠问题 | 推荐方案 |
|---------|-----------|---------|
| 柱状图 | X 轴类别标签重叠 | 旋转 30-45°，或抽稀，或改为水平柱状图 |
| 折线图 | 终点数据标签重叠 | 斜向错落排列 + 竖线指示，或改用 Tooltip |
| 散点图 | 多个密集点的标签混乱 | 力导向避让 + 连线，或只标注异常点 |
| 饼/环图 | 扇区标签重叠 | 改为水平标注条，标签放在外侧 |
| 面积图 | 堆叠区间标签重叠 | 只标注关键时点，其他用颜色区分 |

## 字重与强调层级

字号不是建立视觉层级的唯一手段。适当的字重梯度能更优雅地体现信息重要性，同时避免字号过度膨胀导致的排版失衡。

### 字重的层级设计

`default.json` 中定义的字重规范：

```
h1（标题）：600（半粗）
h2-h6（其他）：400（正常）
```

**建议的三级字重梯度**：

| 重要性 | 字重 | 用途 | 示例 |
|--------|------|------|------|
| 最高（吸引焦点） | 600-700 | 图表主标题、KPI 数值、关键数据标签 | "销售额：¥2.5M"（数字 700） |
| 中级（支持信息） | 500 | 轴标题、副标题、数据类别 | "2024 年 Q1" |
| 最低（辅助信息） | 400 | 刻度标签、图例、脚注、来源 | 轴刻度、图例项 |

### 强调手段的优先级

当需要凸显某个信息时，按以下优先级选择强调手段：

```
优先级：字重 > 颜色 > 字号 > 斜体 > 其他

反面教科书：
错误 1：用字号叠加（最大 28px）来强调一个数字
正确：用字重 700 + 特殊颜色，字号保持 16px

错误 2：同时用粗体、加大、加色三种手段
正确：仅用字重 600 + 凸显色，其他保持一致
```

### 避免超过 3 种字重

超过 3 种字重会导致视觉混乱，难以感知层级。

```
推荐方案（3 种）：
- 400 Regular：脚注、刻度、图例
- 500 Medium：轴标题、副标题
- 600 SemiBold：主标题、重点数据

避免（5+ 种）：
- 300 Light / 400 Regular / 500 Medium / 600 SemiBold / 700 Bold
  → 相邻字重差异不够明显，反而造成困惑
```

### 实际应用示例

**场景：仪表盘 KPI 卡片**

```
┌─────────────────┐
│ 日销售额         │ ← h2: 字号 13px, 400, 灰色 #666666
├─────────────────┤
│  ¥2,500,000     │ ← h1: 字号 20px, 700, 蓝色 #4269d0（用字重和颜色凸显）
├─────────────────┤
│ ↑ 8.5% vs 昨日   │ ← h6: 字号 10px, 400, 绿色 #3ca951（绿色即可表示正增长，无需加粗）
└─────────────────┘
```

**场景：多系列折线图**

```
图表标题："三产品销售趋势"
       ↑ h1: 16px, 600, #1A1A1A

Y 轴标题："销售额（万元）"
       ↑ h3: 11px, 400, #333333

数据线终点标签：
- "产品 A: 45"  → h6: 9px, 400, 蓝色 #4269d0（颜色已体现身份）
- "产品 B: 38"  → h6: 9px, 400, 绿色 #3ca951
- "产品 C: 42"  → h6: 9px, 400, 橙色 #ff725c

如需强调某条线，改为：
- "产品 A: 45"  → h6: 9px, 600, 蓝色 #4269d0（仅提升字重，不加字号）
```

### 检查清单

```
□ 是否使用了不超过 3 种字重（400/500/600）？
□ 最高重要性信息是否用了 600 或以上字重？
□ 是否避免用字号叠加来强调？
□ 颜色强调是否配合字重，而非单独使用？
□ 字重是否与字号成正比（字大时字重不一定重）？
□ 打印效果中字重对比是否仍然明显？
```
