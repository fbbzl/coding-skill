# 用色原则

## 何时使用哪种色板

### Categorical（分类色板）
**使用条件：** 数据类别之间没有内在顺序关系

```
正确：产品 A、产品 B、产品 C → 用 categorical
错误：低、中、高 → 应用 sequential，用 categorical 会误导
```

**建议颜色数量：**
- 2-4 类：颜色区分度最佳
- 5-7 类：可接受，选择差异化最大的颜色
- 8-10 类：谨慎使用，确保有足够视觉差异
- >10 类：合并小类为"其他"，或使用交互式图例

### Sequential（顺序色板）
**使用条件：** 数据有从低到高的自然顺序

**何时用单色相：** 数据的"高低"方向明确且不需要强调极值
**何时用多色相（Viridis）：** 需要更好的感知分辨率、需要色盲友好、需要打印兼容

```
单色相示例：人口密度、年龄分布
多色相示例：科学数据、需要精准读数
```

### Diverging（发散色板）
**使用条件：** 数据围绕一个有意义的中点（零、平均值、目标值）双向分布

```
正确：温度偏差（-10°C 到 +10°C）、同比变化率
错误：从 0 开始的收入分布（应该用 sequential）
```

## 颜色搭配禁忌

1. **红绿对比作为唯一区分**：约 8% 男性无法区分。如需红绿对比，同时使用亮度差异。
2. **彩虹色板（Rainbow）**：感知不均匀，误导数值判断。始终优先使用 Viridis 或类似方案。
3. **过多饱和色**：所有颜色都鲜艳等于没有重点。控制高饱和色 ≤ 3 种/图。
4. **浅色背景上使用黄色**：黄色与白色对比度极低，难以识别。
5. **蓝色文字**：人眼对蓝色细节的分辨力最弱，避免蓝色小字。

## 语义颜色约定

| 颜色 | 常见语义         | 使用场景                 |
|------|------------------|-------------------------|
| 蓝色 | 信任、稳定、主数据 | 默认主要数据系列          |
| 绿色 | 增长、正面、达标   | 目标达成、同比增长        |
| 红色 | 警告、负面、亏损   | 亏损、未达标、需要注意     |
| 橙色 | 注意、中等         | 预警、接近阈值            |
| 灰色 | 中性、背景、参考   | 参考线、基准值、"其他"类别  |
| 紫色 | 高端、特殊         | 对比系列、高亮            |

## 背景与对比度

- 浅色图表：白色背景 (#FFFFFF)，浅灰网格 (#E8E8E8)——默认选择，适合大多数报告
- 深色图表：深色背景 (#1A1A1A)，深灰网格 (#333333)——适合大屏展示和暗色主题
- 文本与背景对比度：正文 ≥ 4.5:1（WCAG AA），大字（≥18px）≥ 3:1
