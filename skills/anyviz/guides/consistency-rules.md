# 设计一致性规则

当一次输出包含多个图表时，以下规则确保视觉统一。

---

## 规则 1：排版层级统一

同一输出中的所有图表，相同角色的文本使用相同的排版规格。

```
检查清单：
□ 所有图表的主标题字号是否一致？（h1: 16px/600）
□ 所有子标题字号是否一致？（h2: 13px/400）
□ 所有轴标题字号是否一致？（h3: 11px/400）
□ 所有刻度标签字号是否一致？（h4: 10px/400）
□ 所有图例文本字号是否一致？（h5: 10px/400）
□ 所有脚注字号是否一致？（h6: 9px/400）
```

## 规则 2：颜色语义统一

同一数据实体在所有图表中使用相同颜色。

```
示例：
如果"华东区"在图表 A 中是 #4269d0（蓝色），
那么它在图表 B、C 中也必须是 #4269d0。
即使图表类型不同（柱状图 vs 折线图 vs 饼图），颜色映射保持一致。
```

**颜色映射表**：当生成多图表时，首先建立一个 `entity → color` 映射表，所有图表引用该表。

## 规则 3：间距统一

同一画布/页面上的多个图表，使用统一的边距和间距。

```
检查清单：
□ 所有图表的 margin 是否一致？
  (top: 40, right: 30, bottom: 50, left: 60)
□ 图表之间的间距是否一致？（24px）
□ 相同宽高比？（默认 1.6:1）
```

## 规则 4：线型与点型统一

同类型的视觉元素使用相同的样式。

```
检查清单：
□ 同数据系列的折线宽度是否一致？（2px）
□ 同数据系列的点大小是否一致？（4px 直径）
□ 网格线样式是否一致？（0.5px, #E0E0E0, dash [2,2]）
□ 参考线样式是否一致？（1px, #ff725c, dash [4,4]）
□ 趋势线样式是否一致？（1.5px, #ff725c, dash [6,3], opacity 0.8）
```

## 规则 5：数值格式统一

同一类数值在所有图表中使用相同格式。

```
检查清单：
□ 小数位数是否一致？（默认 1 位）
□ 百分比格式是否一致？
□ 日期格式是否一致？
□ 单位标注是否一致？（万、亿、%、‰）
□ 千分位分隔符是否一致？
```

## 规则 6：标注风格统一

数据标签、注释、高亮的样式在所有图表中保持一致。

```
检查清单：
□ 数据标签的字体大小和颜色是否一致？
□ 标注线的样式是否一致？
□ 高亮区域的样式是否一致？
□ 提示框（Tooltip）的样式是否一致？
```

---

## 一致性冲突处理优先级

当用户定制与默认规范冲突时：

1. **用户明确指定** → 最高优先级，覆盖默认规范
2. **图表类型特殊要求** → 如饼图不需要轴标签，可省略相关规则
3. **默认规范** → 上述两者未覆盖时，严格遵循默认规范

---

## 规则 7：图表类型一致性

同一份报告中，相同数据角色（如"销售数据""用户增长"）应使用相同的图表类型，确保观看者的心智模型稳定。

### 一致性原则

**一致**：
```
页面 1 中"月度销售"用柱状图
页面 2 中"季度销售"也用柱状图（只是时间粒度不同）
→ 用户快速建立了"销售数据看柱状图"的认知
```

**不一致**：
```
页面 1 中"月度销售"用柱状图
页面 2 中"同样的销售月度数据"用折线图
→ 用户困惑：是否代表不同含义？数据是否可比？
```

### 何时允许混用图表类型

**允许混用** 的前提条件（需同时满足以下条件）：

| 条件 | 说明 |
|------|------|
| 数据角色完全不同 | 如"销售额"（柱状）vs "销售增速"（折线）——不同数据维度 |
| 图表类型选择有明确理由 | 非审美选择，而是数据特性驱使。标注理由在图表标题或脚注 |
| 用户已建立预期 | 通过标题、图例、脚注，用户理解为何切换类型 |
| 不超过 2 种类型 | 报告中不超过 2-3 种图表类型混用，否则显得杂乱 |

**具体场景**：

| 场景 | 推荐 | 理由 |
|------|------|------|
| 销售额同比趋势 | 全用柱状图（同比对比） | 数据角色一致 |
| 销售额 + 利润率同比 | 分别用柱状 + 折线 | 数据单位不同（¥ vs %），需要双轴 |
| 产品 A/B/C 排名对比 | 全用柱状图（改为水平柱避免拥挤） | 只改形式，不改类型 |
| 月度销售 + 年度累计 | 分别用柱状 + 面积图 | 数据含义不同（点数据 vs 累计），可混用 |

### 转换和修改的规范

当因数据特性需要改变图表类型时，记录转换理由：

```
【图表标题】销售额与利润率同比分析（2024 年 1-6 月）

【说明】
销售额（柱状图，左轴）：月度对比，单位：万元
利润率（折线图，右轴）：趋势观察，单位：%
```

## 规则 8：交互行为一致性

多图表报告中，所有交互行为（鼠标悬停、点击、缩放、联动等）应保持一致，让用户建立稳定的交互预期。

### 交互行为的统一规范

| 交互行为 | 统一规范 | 示例 |
|---------|--------|------|
| **Hover 提示框** | 所有图表的 tooltip 样式、出现延迟、内容格式统一 | 背景 #FFFFFF，边框 #D0D0D0，字号 11px，延迟 200ms |
| **高亮效果** | 鼠标悬停时高亮策略一致：高亮该数据点或整个系列 | 柱状图悬停整根柱，折线图悬停整条线，散点图悬停单个点 |
| **点击行为** | 所有可点击元素反应一致：如都支持钻取、都支持选中状态 | 点击图表元素进入详情页，或点击筛选其他图表 |
| **缩放与平移** | 如某图表支持框选缩放，其他同类型图表也应支持 | 避免用户在图表 A 能缩放，图表 B 不能缩放 |
| **联动** | 多图表之间的筛选/选中联动规则明确 | 选中表 A 的某行，表 B 自动筛选相关数据；或不联动，但需明确说明 |
| **颜色高亮** | 高亮时的色彩编码统一：如都用 0.3 opacity 展示非焦点数据 | `deemphased_opacity: 0.25`（参考 stroke 定义） |
| **动画时长** | 所有过渡动画使用一致的时长 | 都用 300ms 缓动，或都无动画（静态报告） |
| **禁用状态** | 不可交互元素的视觉表示统一 | 都用 opacity 0.5 + 鼠标光标 `not-allowed` |

### 多图表联动的设计文档

当报告包含联动时，应明确说明交互流程。

**示例文档**：

```
## 联动规则

### 场景 1：筛选联动
- 用户在"地区选择器"（图例或按钮）中点击某地区
- 所有柱状图、折线图、表格中该地区的数据高亮，其他地区变暗（opacity 0.3）
- 如需进一步隔离，提供"仅显示该地区"的切换按钮

### 场景 2：钻取联动
- 用户点击仪表盘的某个产品类别
- 跳转到产品详情页或展开隐藏的子图表
- 返回按钮恢复原始视图

### 场景 3：时间范围联动
- 用户在上方"时间选择器"中拖选日期范围
- 所有图表自动更新为该时间范围的数据
- 如果某图表不支持该时间粒度，显示"不可用"提示
```

### 键盘与触摸交互

遵循无障碍标准（`guides/accessibility.md`），交互应支持多种输入方式：

```
□ 所有交互都可用键盘（Tab、Enter、方向键）实现
□ 触摸目标大小 ≥ 44×44px
□ 焦点可见（轮廓 ≥ 2px，对比度 ≥ 3:1）
□ Tooltip 不只在 hover 时出现，也可通过点击/聚焦获得
```

## 完整示例：三图表仪表盘一致性核对

以下示例展示如何应用所有 8 条规则，对一个包含销售、利润、区域三个图表的仪表盘进行一致性检查。

### 场景描述

**仪表盘主题**：2024 年上半年销售业绩分析
- 图表 1：销售额按月柱状图（所有产品）
- 图表 2：毛利率按月折线图（所有产品）
- 图表 3：各地区销售额排行柱状图（水平）

### 一致性核对清单与填写

#### **规则 1：排版层级统一**

| 元素 | 目标值 | 图表 1 | 图表 2 | 图表 3 | 状态 |
|------|--------|--------|--------|--------|------|
| 主标题（h1） | 16px, 600 | ✓ 16px | ✓ 16px | ✓ 16px | **通过** |
| 副标题（h2） | 13px, 400 | ✓ 13px | ✓ 13px | ✓ 13px | **通过** |
| 轴标题（h3） | 11px, 400 | ✓ 11px | ✓ 11px | ✓ 11px | **通过** |
| 刻度标签（h4） | 10px, 400 | ✓ 10px | ✓ 10px | ✓ 10px | **通过** |
| 数据标签（h6） | 9px, 400 | ✓ 9px | ✓ 9px | ✓ 9px | **通过** |

**结论**：✓ 全部通过

---

#### **规则 2：颜色语义统一**

建立 `entity → color` 映射表：

| 实体 | 颜色 | 十六进制 | 场景 |
|------|------|---------|------|
| 产品 A | 蓝 | #4269d0 | 柱状、折线 |
| 产品 B | 绿 | #3ca951 | 柱状、折线 |
| 产品 C | 红橙 | #ff725c | 柱状、折线 |
| 华东地区 | 蓝 | #4269d0 | 排行图 |
| 华中地区 | 绿 | #3ca951 | 排行图 |
| 华西地区 | 红橙 | #ff725c | 排行图 |

**核查结果**：

| 检查项 | 图表 1（月度柱状） | 图表 2（月度折线） | 图表 3（排行柱状） | 状态 |
|--------|------------------|------------------|------------------|------|
| 产品 A 用 #4269d0？ | ✓ | ✓ | N/A | **通过** |
| 产品 B 用 #3ca951？ | ✓ | ✓ | N/A | **通过** |
| 产品 C 用 #ff725c？ | ✓ | ✓ | N/A | **通过** |
| 地区色与产品一致？ | N/A | N/A | ✓ 华东=#4269d0 | **通过** |

**结论**：✓ 全部通过，数据实体的颜色映射一致

---

#### **规则 3：间距统一**

| 间距项 | 标准值 | 图表 1 | 图表 2 | 图表 3 | 备注 |
|--------|--------|--------|--------|--------|------|
| 上边距 | 48px | ✓ | ✓ | ✓ | 标题到图表 |
| 右边距 | 36px | ✓ | ✓ | ✓ | 图表到右边界 |
| 下边距 | 56px | ✓ | ✓ | ✓ | 图表到图例 |
| 左边距 | 56px | ✓ | ✓ | ✓ | Y 轴标签到图表 |
| 图表间距 | 28px | ✓ | ✓ | ✓ | 三图纵向间距 |
| 宽高比 | 1.6:1 | ✓ | ✓ | 1:0.4 | 第 3 图为水平柱，特殊比例 |

**结论**：✓ 全部通过。注：图表 3 因类型特殊（水平柱），宽高比可调整但仍应遵循 grid 对齐

#### **规则 4：线型与点型统一**

| 视觉元素 | 标准 | 图表 1 | 图表 2 | 图表 3 | 状态 |

#### **规则 7：图表类型一致性**

## 与无障碍标准的交集

多图表的一致性检查应同时遵循 `guides/accessibility.md` 的要求：

```
□ 所有图表是否都通过了对比度测试？
□ 是否有冗余编码（颜色 + 形状 / 标签）？
□ 红绿语义是否加了符号？
□ 关键数据是否有 alt text？
□ 交互是否支持键盘 + 触摸？
```

一致性检查与无障碍检查可视为两个正交的维度，应独立但互补地执行。
