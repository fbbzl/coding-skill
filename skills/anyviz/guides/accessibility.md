# 无障碍标准（Accessibility Guide）

> 可访问的图表不是给少数人的额外功能，而是更清晰、更稳健的设计。本指南把 `aesthetics/default.json` 中 `accessibility` 字段的约束展开为可执行规范。生成可视化时应默认遵循，除非调用者明确放弃某项。

## 为什么重要

约 8% 的男性和 0.5% 的女性有某种色觉障碍；更多人在弱光、低质量屏幕、打印灰度或强光环境下查看图表。任何只靠颜色传达的信息，对这些场景都会丢失。无障碍设计的核心是**冗余编码**：每个关键区分至少有两种视觉通道支撑（颜色 + 形状 / 颜色 + 标签 / 颜色 + 纹理 / 颜色 + 位置）。

---

## 1. 色彩对比度

### 文本对比度（WCAG 2.1）

| 内容类型 | 最低对比度 | 标准 |
|----------|-----------|------|
| 正文（< 18px 或 < 14px 加粗） | 4.5:1 | AA |
| 大字（≥ 18px 或 ≥ 14px 加粗） | 3:1 | AA |
| 正文（追求更高标准） | 7:1 | AAA |
| 图形元素 / UI 边界 | 3:1 | AA |

`default.json` 中文本颜色（#1A1A1A 主 / #555555 次 / #666666 三级 / #888888 弱）在白底（#FFFFFF）上均满足 AA。弱文本 #888888 在白底约 3.5:1，仅适合脚注等非关键信息，不要用于核心数据标签。

### 数据色与背景

- 数据元素（柱、线、点）与背景对比度 ≥ 3:1，否则在弱视或低质屏幕下不可见。
- 浅色背景上避免纯黄（#FFEB3B 类），与白对比极低；如需用黄，加深至 #E8B33F 或配深色描边。
- 蓝色细节分辨力最弱，避免蓝色小字（< 10px）。

## 2. 色盲友好

### 默认色板已验证

`default.json` 的 categorical 10 色色板经 ColorBrewer 验证，在以下三类色觉障碍模拟下均可区分：

- **红绿色盲**（Protanopia / Deuteranopia，最常见）：避免仅靠红-绿区分相邻类别
- **蓝黄色盲**（Tritanopia，较少见）：避免仅靠蓝-黄区分
- **全色盲**（Achromatopsia）：依赖灰度明度差，色板设计为灰度下仍可分辨（`grayscale_distinguishable: true`）

### 红绿语义的陷阱

「红跌绿涨」「红负绿正」是最常见的无障碍雷区——红绿色盲恰好无法区分这两色。处理方式：

1. 保留红绿色作为习惯，但**叠加形状/符号**：涨 ▲ / 跌 ▼，正 + / 负 −
2. 或改用**蓝-橙发散色板**（diverging），蓝橙对所有色觉类型都可区分
3. 烛台图、瀑布图、热力图、KPI 卡片是高发场景，务必加冗余编码

### 验证方法

生成多色图表后，建议在脑中或工具中过一遍色盲模拟。可用色板间的**明度差**（lightness）作为兜底——即使色相不可分，明度阶梯也能区分。

## 3. 冗余编码（不只靠颜色）

每种图表类型的冗余编码策略：

| 图表 | 仅靠颜色的风险 | 冗余编码方案 |
|------|---------------|-------------|
| 折线图（多系列） | 色盲无法区分线 | 不同 dash 模式（实线/虚线/点线）+ 终点直接标注 |
| 柱状/堆叠图 | 堆叠段难分 | 白色描边分隔 + 段内/图例标签 + 纹理（可选） |
| 饼/环图 | 扇区颜色相近 | 直接标注名称+百分比，不只靠图例 |
| 散点（分类） | 类别色难分 | 不同形状（圆/三角/方/十字）+ 颜色 |
| 热力图 | 顺序色阶难读 | 单元格内显示数值 + 选用 Viridis（明度单调） |
| 烛台/瀑布 | 红绿不可分 | ▲▼ 符号 / 实虚边框 + 数值标签 |
| 地图（choropleth） | 区域色难比 | 数值标注 + 图例分级清晰 + Viridis |
| 桑基/冲积/弦 | 流带颜色混 | 标签完整 + 纹理可选 + 数据表视图 |

**形状编码梯度**（散点等推荐顺序）：圆 ● → 三角 ▲ → 方 ■ → 菱 ◆ → 十字 ✕ → 星 ★。前 4 种区分度最高。

**线型编码梯度**：实线 → 长虚线 [6,3] → 短虚线 [3,3] → 点线 [1,2] → 点划线 [6,3,1,3]。

## 4. 文本替代（Alt Text）

每张图表应提供文本替代，供屏幕阅读器和无法加载图像的场景使用。

`default.json` 提供模板：
```
{chart_type} 图表，展示 {data_description}。X 轴：{x_axis_label}，Y 轴：{y_axis_label}。
```

好的 alt text 分三层：

1. **类型与主题**：「折线图，展示 2024 年三个产品的月度销售额」
2. **关键趋势/结论**：「产品 A 全年稳步上升，产品 C 在 Q3 出现明显下滑」
3. **数据出口**：复杂图表附带数据表（`<table>`）或可下载 CSV，让读者自行探索精确值

避免无信息量的 alt text，如「图表」「图片」「sales chart」。

## 5. 交互无障碍

交互式图表（D3 / ECharts / Plotly）额外要求：

- **键盘可达**：可聚焦、可用方向键在数据点间移动、Enter/空格触发，不依赖鼠标悬停
- **焦点可见**：聚焦元素有清晰描边（≥ 2px，对比度 ≥ 3:1）
- **触摸目标**：交互热区 ≥ 44×44px（`min_touch_target_px: 44`），小数据点需扩大点击区
- **Tooltip 不只靠 hover**：信息也应能通过点击/聚焦获得；tooltip 文本对比度达标
- **动画可关闭**：尊重 `prefers-reduced-motion`，为前庭敏感用户提供静态版本；避免闪烁（每秒 > 3 次闪烁有诱发癫痫风险）
- **不依赖颜色的状态**：选中/高亮/禁用状态除颜色外用边框、图标、透明度区分

---

## 6. 字号与可读性

- 核心数据标签 ≥ 9px（`default.json` H6），但屏幕展示建议 ≥ 11px
- 移动端按 `responsive.breakpoints.mobile` 的 `font_scale: 0.85` 后仍不应低于 9px，必要时减少标签密度而非缩小字号
- 行高 ≥ 1.3（`default.json` 各层级已设 1.3）
- 避免全大写长文本（降低可读性），避免居中对齐的多行正文

---

## 7. 生成时的无障碍检查清单

多图表或正式输出前过一遍：

```
□ 文本与背景对比度 ≥ 4.5:1（大字 ≥ 3:1）？
□ 数据元素与背景对比度 ≥ 3:1？
□ 多类别是否有颜色之外的第二编码（形状/线型/标签）？
□ 红绿语义是否加了符号或改用蓝橙发散？
□ 顺序数据是否用了 Viridis 或明度单调的色板？
□ 是否提供了 alt text（类型+趋势+数据出口）？
□ 复杂图是否附数据表/CSV？
□ 交互图是否键盘可达、焦点可见、触摸目标 ≥ 44px？
□ 动画是否可关闭、无高频闪烁？
□ 关键标签字号是否 ≥ 9px（屏幕建议 ≥ 11px）？
```
